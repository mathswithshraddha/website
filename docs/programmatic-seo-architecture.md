# Programmatic SEO Architecture

## Goals

This implementation turns the site into a curated programmatic SEO system for olympiad maths, mental maths, singapore maths, and premium online enrichment without creating open-ended doorway pages.

The system is designed to:

- scale through structured data, not thin templates
- keep indexation controlled with `dynamicParams = false`
- preserve existing route equity by leaving current pages intact
- add international-ready, online-first landing pages
- support future expansion by extending datasets instead of duplicating page files

## Route Model

The pSEO engine lives in [app/[...slug]/page.tsx](/home/vedantchalke36/maths-with-shraddha/app/[...slug]/page.tsx), powered by the route registry in [lib/seo/routes.ts](/home/vedantchalke36/maths-with-shraddha/lib/seo/routes.ts) and the datasets in [data/seo/programmatic.ts](/home/vedantchalke36/maths-with-shraddha/data/seo/programmatic.ts).

Supported route families:

1. `exam + city`
2. `exam + grade`
3. `exam + city + grade`
4. `parent intent`
5. `comparison`
6. `local SEO`
7. `international`
8. `online-first`

The route list is explicitly generated. Unknown combinations do not render and are not indexed.

## Why This Is SEO-Safe

- No unbounded slug parser: every page must exist in the curated route registry.
- No dynamic low-value permutations: city-grade combinations are limited to higher-intent locations and topics.
- No duplicate canonicals: each route has a single canonical URL generated by the metadata engine.
- No indexable fallback junk: `dynamicParams = false` and `notFound()` enforce hard boundaries.

## Metadata Engine

[lib/seo/metadata.ts](/home/vedantchalke36/maths-with-shraddha/lib/seo/metadata.ts) centralizes:

- title generation
- description generation
- canonical URLs
- robots directives
- Open Graph
- Twitter metadata

The root layout now uses a single site metadata definition instead of page-by-page ad hoc values.

## Schema System

[lib/seo/schema.ts](/home/vedantchalke36/maths-with-shraddha/lib/seo/schema.ts) generates:

- `EducationalOrganization`
- `WebPage`
- `Course`
- `FAQPage`
- `BreadcrumbList`
- `LocalBusiness`

JSON-LD is rendered through [components/seo/JsonLd.tsx](/home/vedantchalke36/maths-with-shraddha/components/seo/JsonLd.tsx).

## Content Strategy

[lib/seo/content.ts](/home/vedantchalke36/maths-with-shraddha/lib/seo/content.ts) assembles page content from:

- exam intent
- grade stage
- location context
- regional profile
- parent-intent angle
- comparison framing

Every page gets:

- unique H1
- unique summary copy
- exam- or location-specific sections
- FAQs
- internal links
- trust points
- conversion CTAs

The copy is still modular, but the modules are intent-based rather than filler-based.

## Internal Linking

Internal linking is generated from the same route intelligence:

- related exam pages
- online versions of related offers
- nearby location pages
- central resources hub

This gives the site cluster structure without requiring manual maintenance for every future landing page.

## Resource Hub / Topical Authority

A dedicated topical hub is added at:

- [app/resources/page.tsx](/home/vedantchalke36/maths-with-shraddha/app/resources/page.tsx)
- [app/resources/[slug]/page.tsx](/home/vedantchalke36/maths-with-shraddha/app/resources/[slug]/page.tsx)

The content data is stored in [data/resources.ts](/home/vedantchalke36/maths-with-shraddha/data/resources.ts).

This creates a non-sales topical authority layer for searches such as:

- best maths olympiads
- mental maths benefits
- SASMO preparation at home
- singapore maths problem solving
- online maths enrichment
- olympiad maths and STEM

## International Strategy

International pages are generated from region data in `seoRegions`.

The system supports:

- regional locale codes
- country-aware copy
- spelling differences like `maths` vs `math`
- spelling differences like `programme` vs `program`
- timezone-aware positioning in copy
- online-first messaging for global families

For broader i18n later, this data model can be moved under a locale-aware route structure without rewriting page logic.

## Sitemap and Crawl Control

[app/sitemap.ts](/home/vedantchalke36/maths-with-shraddha/app/sitemap.ts) now includes:

- static pages
- programmatic SEO pages
- resource articles

[app/robots.txt](/home/vedantchalke36/maths-with-shraddha/app/robots.txt) continues to block non-public surfaces and points crawlers to the unified sitemap.

## How To Scale

To add more pages safely:

1. Extend the datasets in `data/seo/programmatic.ts`.
2. Adjust route-generation rules in `lib/seo/routes.ts` if needed.
3. Add supporting content logic in `lib/seo/content.ts` only when a new route family needs distinct messaging.

Do not add free-form slug parsing. Keep route creation curated.
